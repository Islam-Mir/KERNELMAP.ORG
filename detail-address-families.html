DESCRIZIONE:

Le Address Families (Famiglie di Indirizzi) sono il sistema di classificazione degli indirizzi di rete nel kernel Linux. Ogni famiglia di indirizzi rappresenta un diverso tipo di protocollo di rete e definisce come gli indirizzi vengono rappresentati, validati e utilizzati per il routing e la comunicazione.

CONFRONTO:

Immagina le famiglie di indirizzi come diversi linguaggi di comunicazione: IPv4 √® come l'italiano tradizionale, IPv6 come l'italiano moderno con pi√π parole, mentre Unix domain sockets sono come parlare direttamente con qualcuno nella stessa stanza senza bisogno di telefono.

üîç Descrizione Tecnica

Le address families definiscono il formato e la semantica degli indirizzi di rete utilizzati nel networking del kernel. Ogni famiglia ha le proprie strutture dati, funzioni di validazione e protocolli di comunicazione associati.

‚Ä¢ AF_INET (IPv4) - Indirizzi a 32-bit per reti TCP/IP classiche

‚Ä¢ AF_INET6 (IPv6) - Indirizzi a 128-bit per reti moderne espandibili

‚Ä¢ AF_UNIX - Comunicazione tra processi sulla stessa macchina

‚Ä¢ Interfaccia principale per il socket layer e protocolli di rete

üìã Esempi Pratici

Le famiglie di indirizzi sono fondamentali per ogni comunicazione di rete in Linux:

üîç Connessione TCP IPv4

1. Applicazione chiama socket(AF_INET, SOCK_STREAM, 0)

2. Kernel crea socket con indirizzo IPv4

3. connect() stabilisce connessione TCP

4. Dati trasferiti attraverso indirizzo IPv4

üìÑ Strutture Dati Chiave

  • struct sockaddr_in - Struttura indirizzo IPv4 (porta + indirizzo 32-bit)
  • struct sockaddr_in6 - Struttura indirizzo IPv6 (porta + indirizzo 128-bit)
  • struct sockaddr_un - Struttura per Unix domain socket (path filesystem)
  • struct addrinfo - Informazioni risoluzione nomi generiche

  • $ ss -tuln - Lista socket TCP/UDP in ascolto
  • $ netstat -i - Statistiche interfacce di rete

üéØ Caso Reale: Browser Web

Quando fai $ curl https://www.example.com:

üõ†Ô∏è API Socket

Queste sono le funzioni principali per lavorare con le famiglie di indirizzi:

üìÇ Creazione e Configurazione Socket

int socket(int domain, int type, int protocol);
Crea un socket di una famiglia specifica (AF_INET, AF_INET6, AF_UNIX).

int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
Associa un indirizzo locale al socket (porta + indirizzo IP).

int listen(int sockfd, int backlog);
Mette il socket in ascolto per connessioni in entrata.

int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
Accetta una connessione e restituisce un nuovo socket per la comunicazione.

üìä Connessione e Comunicazione

int connect(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
Stabilisce una connessione con un indirizzo remoto.

ssize_t send(int sockfd, const void *buf, size_t len, int flags);
Invia dati attraverso un socket connesso.

ssize_t recv(int sockfd, void *buf, size_t len, int flags);
Riceve dati da un socket connesso.

üíΩ Risoluzione Nomi e Utility

int getaddrinfo(const char *node, const char *service, const struct addrinfo *hints, struct addrinfo **res);
Risolve un nome host in indirizzi IP (supporta IPv4/IPv6).

const char *inet_ntop(int af, const void *src, char *dst, socklen_t size);
Converte un indirizzo binario in stringa leggibile (IPv4/IPv6).

üîç Esempi di Utilizzo Pratico

Come utilizzare le famiglie di indirizzi nella programmazione di rete:

  1. socket(AF_INET, SOCK_STREAM, 0) - Crea socket TCP IPv4
  2. bind(sock, (struct sockaddr*)&addr, sizeof(addr)) - Associa indirizzo locale
  3. connect(sock, (struct sockaddr*)&server, sizeof(server)) - Connette al server
  4. send(sock, buffer, size, 0) - Invia dati attraverso la connessione

Queste API permettono di scrivere codice di rete portabile che funziona con diverse famiglie di indirizzi.

üíª Come funziona la risoluzione e l'uso degli indirizzi

üì• Nome Host

Processo eseguito: L'applicazione fornisce un nome host, il resolver DNS converte il nome in un indirizzo IP della famiglia appropriata (IPv4 o IPv6), e il socket viene creato con la famiglia di indirizzi corrispondente.

üîÑ Fasi della comunicazione di rete:

  1. 1. Risoluzione Nome - DNS converte nome host in indirizzo IP

  4. 4. Scambio Dati - send()/recv() trasferiscono dati attraverso la connessione
üéØ Risultato: I dati vengono trasferiti in modo affidabile attraverso la rete utilizzando la famiglia di indirizzi appropriata, con supporto automatico per IPv4/IPv6 dual-stack.
