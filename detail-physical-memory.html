üìã Strutture Dati Chiave

Il kernel utilizza strutture specializzate per tracciare la memoria fisica:

Page Frame Number (PFN)

  • Indirizzi fisici - Numeri che identificano univocamente ogni pagina fisica
  • Page flags - Stato pagina (libera, riservata, swappable)
  • Reference counting - Quanti processi usano questa pagina
  • LRU lists - Liste per page replacement algorithms

Buddy System

  • Free lists - Liste di blocchi liberi per ogni dimensione (1,2,4,8... pagine)
  • Splitting - Suddivisione blocchi grandi in pi√π piccoli
  • Merging - Ricombinazione blocchi adiacenti liberi
  • Fragmentation control - Minimizzazione frammentazione esterna

Memory Map

  • E820 map - Mappa memoria fornita dal BIOS/firmware
  • Reserved areas - Memoria riservata per dispositivi, kernel text/data
  • Available memory - Memoria utilizzabile dal sistema operativo
  • NUMA nodes - Suddivisione memoria per architetture NUMA

üéØ Caso Reale: Allocazione Pagina

Quando un programma richiede memoria:

  1. Buddy allocator cerca blocco libero adatto
  2. Suddivide blocchi grandi se necessario
  3. Marca pagina come allocata
  4. Restituisce indirizzo fisico al virtual memory manager
  5. Virtual memory mappa indirizzo logico a fisico

üíæ Buddy System
Cerca in free_list[0]
Se vuoto, suddivide blocco da free_list[1]
Algoritmo buddy system: free_list[0] ‚Üí free_list[1] ‚Üí free_list[2] ‚Üí ... ‚Üí splitting

üîÑ Processo di allocazione:

  1. T=0.001ms: Controllo free_list[0] - Lista pagine singole (order 0)
  2. T=0.002ms: Splitting necessario - Lista vuota, prendi da free_list[1] (2 pagine)
  3. T=0.003ms: Suddivisione blocco - Spezza blocco 2 pagine in due blocchi 1 pagina
  4. T=0.004ms: Allocazione completata - Una pagina allocata, una torna in free_list[0]
  5. T=0.005ms: Ritorno indirizzo - PFN restituito al virtual memory manager
