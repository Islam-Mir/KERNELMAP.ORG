üìã Esempi Pratici

Quando apri un file di testo con un editor, ecco cosa succede nei meccanismi di accesso:

üîç Apertura del File

1. Il processo chiama open() con il pathname e i flags desiderati

2. Il kernel verifica i permessi di accesso basati su UID/GID

3. Viene creato un file descriptor e restituito al processo

4. Il file descriptor viene associato alla tabella dei file aperti del processo

üìÑ Strutture Dati Chiave

  • File Descriptor - Numero intero che identifica un file aperto nel contesto di un processo
  • File Table Entry - Struttura che mantiene lo stato di un file aperto (posizione, flags, riferimenti)
  • Inode - Struttura che contiene i metadati del file sul disco
  • Dentry - Cache delle associazioni nome-inode per lookup efficienti

üíª Nel Computer: Come Vederlo?

Comandi Linux per esplorare l'accesso ai file:

  • lsof - Lista tutti i file aperti dai processi
  • fuser -v file.txt - Mostra quali processi stanno usando un file
  • ls -l /proc//fd/ - Vede i file descriptor di un processo
  • strace -e open,read,write programma - Traccia le system call di accesso file

üéØ Caso Reale: Lettura di un File

Quando fai cat file.txt:

  1. Il comando cat chiama open() per aprire il file
  2. Ottiene un file descriptor (tipicamente 3)
  3. In un ciclo chiama read() fino a EOF
  4. I dati letti vengono scritti su stdout (file descriptor 1)
  5. Alla fine chiama close() per rilasciare il file descriptor

  • O_WRONLY - Solo scrittura
  • O_RDWR - Lettura e scrittura
  • O_APPEND - Scrittura in append
  • O_CREAT - Crea il file se non esiste

üõ†Ô∏è API Principali

Queste sono le funzioni che ogni programmatore usa quotidianamente per accedere ai file:

üìÇ Gestione File Base

Apre un file e restituisce un file descriptor. Verifica permessi e inizializza strutture dati.

Legge fino a 'count' byte dal file descriptor nella memoria del processo.

Scrive fino a 'count' byte dalla memoria del processo nel file.

int close(int fd);
Chiude il file descriptor e rilascia le risorse associate.

üéØ Controllo e Posizionamento

Cambia la posizione corrente nel file per letture/scritture successive.

int fcntl(int fd, int cmd, ...);
Controlla propriet√† del file descriptor (flags, locks, etc.).

int flock(int fd, int operation);

üíΩ Operazioni Avanzate

Mappa un file nella memoria virtuale, permettendo accessi come array.

Sincronizza i dati bufferizzati con il dispositivo di storage.

üîç Esempi di Utilizzo Quotidiano

Quando copi un file con cp:

  1. open(sorgente, O_RDONLY) - Apre file sorgente in sola lettura

  3. Ciclo: read() chunk dalla sorgente, write() nella destinazione
  4. close() entrambi i file descriptor

Tutto questo passa attraverso i meccanismi di accesso ai file, che gestiscono permessi, buffering, e sincronizzazione!
