‚ö° Flusso dell'interrupt di rete:

  1. T=0.001ms: Hardware interrupt - NIC genera MSI interrupt sulla linea assegnata
  2. T=0.002ms: CPU context save - Processore salva stato corrente e passa a kernel mode
  3. T=0.003ms: Top-half ISR - Handler riconosce interrupt rete, disabilita ulteriori interrupt
  4. T=0.005ms: Packet processing - Driver copia pacchetto da buffer NIC a memoria kernel
  5. T=0.008ms: Bottom-half scheduling - Schedula softIRQ per elaborazione protocollo
  6. T=0.010ms: Return to user space - CPU torna al processo interrotto
  7. T=0.050ms: Protocol processing - SoftIRQ elabora TCP/IP stack e consegna dati all'applicazione
