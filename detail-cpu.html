üé¨ Funzionamento della CPU

üíª Come funziona il multitasking della CPU

üì• Timer Interrupt
Timer hardware interrompe processo corrente ogni 1-10ms
‚ö°
‚öôÔ∏è Context Switch
Kernel salva stato corrente e carica nuovo processo
Processo eseguito: Il timer hardware genera interrupt periodici che permettono al kernel di interrompere processi, salvare il loro stato (registri, stack pointer, memoria virtuale) e selezionare il prossimo processo da eseguire attraverso lo scheduler CFS.

üîÑ Ciclo di scheduling CPU:

  1. 1. Timer Tick - Interrupt timer ogni millisecondi
  2. 2. Preemption Check - Verifica se processo deve essere interrotto
  3. 3. State Save - Salva registri e stack del processo corrente
  4. 4. Scheduler Run - Seleziona prossimo processo dalla runqueue
  5. 5. Context Restore - Carica registri e memoria del nuovo processo
  6. 6. Resume Execution - Processo continua dall'istruzione successiva
üéØ Risultato: Multipli processi sembrano eseguire simultaneamente grazie al rapidissimo context switching (milioni di operazioni al secondo), garantendo fairness e responsiveness del sistema.
