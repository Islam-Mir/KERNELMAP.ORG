üìã Struttura del Filesystem

Il filesystem Linux organizza file e directory in una struttura gerarchica:

üå≥ Struttura Gerarchica

  • Root directory (/) - Radice del filesystem
  • Directory standard - /bin, /etc, /home, /usr, /var
  • Directory utente - /home/username/
  • Directory di sistema - /proc, /sys, /dev

üìÑ Tipi di File

  • File regolari - Contengono dati (testo, binari, immagini)
  • Directory - Contengono riferimenti ad altri file
  • Device files - Rappresentano dispositivi hardware
  • Named pipes - Canali di comunicazione tra processi
  • Socket files - Endpoint di comunicazione di rete
  • Symbolic links - Puntatori ad altri file

üîê Permessi Unix

  • Proprietario (user) - rwx (read, write, execute)
  • Gruppo (group) - rwx
  • Altri (others) - rwx
  • Bit speciali - setuid, setgid, sticky bit

üéØ Caso Reale: Creazione File

Quando fai touch nuovo.txt:

  1. Kernel verifica permessi directory
  2. Alloca inode per il nuovo file
  3. Crea directory entry nella directory padre
  4. Imposta metadata di default (permessi, timestamp)
  5. Aggiorna cache del filesystem

üîß Gestione Avanzata

  • Hard links - Multiple directory entries per stesso inode
  • Soft links - Puntatori simbolici ad altri file
  • File descriptors - Handle numerici per file aperti
  • Directory streams - Iteratori per contenuto directory

üîç Esempi di Utilizzo Quotidiano
