üîÑ Come funziona l'ottimizzazione I/O:

  1. T=0.001ms: System call - read() arriva al VFS layer
  2. T=0.002ms: Cache lookup - Page cache verifica presenza dati
  3. T=0.003ms: Cache miss - Dati non in memoria, richiesta I/O
  4. T=0.005ms: I/O scheduling - CFQ scheduler ottimizza ordine richieste
  5. T=0.008ms: Read-ahead - Kernel anticipa lettura blocchi successivi
  6. T=0.010ms: DMA setup - Controller configura trasferimento diretto
  7. T=0.050ms: Transfer complete - Dati in memoria dopo ~40Œºs
  8. T=0.051ms: Second read - Cache hit, risposta immediata
üéØ Risultato: Prima lettura: ~50Œºs (I/O fisica), Seconda lettura: ~0.5Œºs (cache hit). Il sistema ottimizza automaticamente l'accesso ai dati!

üîÑ

üöÄ

DMA e Zero-Copy

DMA permette trasferimenti diretti memoria-dispositivo senza CPU. Zero-copy elimina copie inutili per massimizzare prestazioni.
