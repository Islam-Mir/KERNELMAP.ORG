üìã Esempi Pratici

Quando apri un file su disco come /dev/sda, ecco cosa succede nel Device Access:

üîç Risoluzione Device

1. Il VFS riconosce "/dev/sda" come device file speciale

2. Cerca il driver appropriato nella device table

3. Verifica permessi di accesso al dispositivo

4. Crea connessione tra processo e driver

üìÑ Strutture Dati Chiave

  • Device driver - Modulo kernel che gestisce dispositivo specifico
  • Device file - File speciale in /dev che rappresenta dispositivo
  • Major/Minor numbers - Identificatori univoci del dispositivo
  • File operations - Tabella funzioni del driver

üéØ Caso Reale: Lettura da Disco

Quando fai dd if=/dev/sda of=backup.img bs=1M count=100:

  1. Kernel identifica /dev/sda come block device
  2. Carica driver appropriato (es. ahci, nvme)
  3. Traduce richiesta logica in comandi hardware
  4. Coordina DMA per trasferimento dati
  5. Restituisce dati al processo user-space

üîç Esempi di Utilizzo Quotidiano

Quando fai un backup del disco:

  1. open("/dev/sda", O_RDONLY) - Apre dispositivo disco
  2. open("backup.img", O_WRONLY|O_CREAT) - Crea file destinazione
  3. Ciclo: read() dal device, write() nel file
  4. close() entrambi i file descriptors

Il kernel astrae completamente l'hardware sottostante!
