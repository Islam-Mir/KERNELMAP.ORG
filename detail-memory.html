üß† MEMORIA

DESCRIZIONE:

Il sottosistema MEMORY del kernel Linux gestisce l'allocazione, la gestione e l'ottimizzazione della memoria fisica e virtuale del sistema. Questo componente critico coordina l'accesso alla RAM, gestisce la memoria virtuale, implementa meccanismi di caching avanzati e garantisce l'isolamento tra processi attraverso la memoria virtuale.

CONFRONTO:

Immagina il sottosistema MEMORY come un magazziniere esperto in un grande deposito: organizza gli scaffali (RAM fisica), tiene traccia di ogni oggetto (dati), ottimizza lo spazio (compression), e garantisce che ogni cliente (processo) abbia accesso solo ai propri beni (isolamento memoria).

üîç Descrizione Tecnica

Il memory management implementa un'architettura complessa che include virtual memory, paging, swapping, memory mapping e meccanismi di protezione. Utilizza algoritmi sofisticati per l'allocazione dinamica, il garbage collection implicito e l'ottimizzazione delle prestazioni.

üõ†Ô∏è API del Memory Management

Le API del memory management forniscono controllo completo sulla memoria fisica e virtuale:

üìÇ Memory Allocation

void *malloc(size_t size);
Allocazione memoria dinamica heap. Chiama brk() o mmap() internamente.

void *mmap(void *addr, size_t length, int prot, int flags, int fd, off_t offset);
Mappa file o memoria anonima nello spazio virtuale del processo.

void *kmalloc(size_t size, gfp_t flags);
Allocazione memoria kernel. Gestisce slab allocator e zone memoria.

void free(void *ptr);
Deallocazione memoria. Ritorna memoria al sistema per riuso.

üìä Virtual Memory Control

int mprotect(void *addr, size_t len, int prot);
Modifica protezione pagine virtuali (read/write/execute).

int madvise(void *addr, size_t length, int advice);
Consigli al kernel su uso memoria (sequential/random/dontneed).

int mlock(const void *addr, size_t len);
Blocca pagine in memoria fisica per evitare swapping.

üíΩ Memory Information

int getrusage(int who, struct rusage *usage);
Ottiene statistiche uso risorse incluse memoria massima utilizzata.

long sysconf(int name);
Ottiene configurazione sistema (_SC_PAGESIZE, _SC_PHYS_PAGES).

size_t malloc_usable_size(void *ptr);
Restituisce dimensione effettiva blocco memoria allocato.

üîç Esempi di Utilizzo Pratico

Come utilizzare le API memory management nelle applicazioni:

  1. ptr = malloc(1024) - Alloca 1KB di memoria heap
  2. mmap(NULL, size, PROT_READ|PROT_WRITE, MAP_PRIVATE|MAP_ANONYMOUS, -1, 0) - Alloca memoria virtuale
  3. mlock(buffer, size) - Impedisce swapping di dati critici
  4. madvise(ptr, size, MADV_SEQUENTIAL) - Ottimizza per accesso sequenziale

Queste API forniscono controllo granulare sulla memoria fisica e virtuale del sistema.

üé¨ Funzionamento del Memory Management

üíª Come funziona la gestione della memoria

üì• Richiesta Allocazione
Applicazione richiede memoria (malloc, mmap)
‚ö°
‚öôÔ∏è Gestione Virtuale
Kernel assegna spazio virtuale e gestisce mapping fisico
Processo eseguito: Il memory management riceve richieste di allocazione, assegna spazio virtuale ai processi, gestisce il mapping tra indirizzi virtuali e fisici attraverso page table, implementa demand paging per caricamento lazy delle pagine, e garantisce isolamento tra processi attraverso memoria virtuale protetta.

üîÑ Ciclo di vita di una richiesta memoria:

  1. 1. Virtual Allocation - Assegnazione spazio indirizzo virtuale al processo
  2. 2. Page Table Setup - Configurazione page table per traduzione indirizzi
  3. 3. Demand Loading - Caricamento pagine fisiche solo su primo accesso (page fault)
  4. 4. Physical Mapping - Allocazione pagine fisiche e aggiornamento PTE
  5. 5. Access Control - Applicazione permessi di accesso (read/write/execute)
  6. 6. Cleanup - Deallocazione e ritorno memoria al sistema su free()
üéØ Risultato: Ogni processo ottiene uno spazio memoria privato e isolato, con allocazione dinamica trasparente e gestione automatica delle risorse fisiche limitate.
