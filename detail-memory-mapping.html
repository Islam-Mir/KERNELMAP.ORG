üìã Esempi Pratici

Quando usi mmap() per mappare un file in memoria, ecco cosa succede:

üîç Memory-Mapped File

1. Il kernel valida i parametri di mmap() (file descriptor, offset, lunghezza)

2. Alloca strutture vm_area_struct per rappresentare il mapping

3. Popola le tabelle di pagine con entry che puntano ai dati del file

4. Restituisce un indirizzo virtuale al processo chiamante

üìÑ Strutture Dati Chiave

  • vm_area_struct - Rappresenta una regione di memoria virtuale mappata, con indirizzi, permessi, e operazioni associate

  • mm_struct - Descrive lo spazio di indirizzamento completo di un processo, contenente la gerarchia delle tabelle di pagine
  • address_space - Gestisce le pagine associate a un file, coordinando tra cache e storage

üéØ Caso Reale: Database Memory-Mapped

Quando un database usa mmap() per file di dati:

  1. Il database chiama mmap() per mappare l'intero file in memoria
  2. Il kernel crea PTE che puntano direttamente ai blocchi del file
  3. Le query accedono ai dati come se fossero in memoria normale
  4. Il page fault carica automaticamente i dati dal disco quando necessario
  5. Le modifiche vengono scritte automaticamente su disco dal kernel

üó∫Ô∏è Tipi di Memory Mapping

  • File-backed mapping - Mappa contenuto di file direttamente in memoria (mmap() con fd)
  • Anonymous mapping - Alloca memoria senza file sottostante (MAP_ANONYMOUS)
  • Shared mapping - Modifiche visibili ad altri processi (MAP_SHARED)
  • Private mapping - Modifiche private con copy-on-write (MAP_PRIVATE)
  • Huge pages - Mappature con pagine di grandi dimensioni per performance

‚ú® Caratteristiche

  • Zero-copy I/O - Accesso diretto ai dati senza copie aggiuntive
  • Demand paging - Caricamento lazy delle pagine su page fault
  • Shared memory - Comunicazione efficiente tra processi
  • Memory protection - Controllo granulare di lettura/scrittura/esecuzione
  • TLB optimization - Cache efficiente delle traduzioni indirizzo
