üõ†Ô∏è API Principali

Queste sono le funzioni che ogni programmatore usa quotidianamente, spesso senza sapere che parlano con il VFS:

üìÇ Gestione File Base

Apre un file e restituisce un file descriptor. Il VFS risolve il pathname e chiama il filesystem specifico.

ssize_t read(int fd, void *buf, size_t count);
Legge dati dal file nel buffer. Il VFS coordina con il page cache per evitare accessi disco inutili.

ssize_t write(int fd, const void *buf, size_t count);
Scrive dati nel file. I dati vanno prima nel page cache, poi vengono scritti su disco in background.

int close(int fd);
Chiude il file e rilascia il file descriptor. Forza la scrittura di dati buffered.

üìä Metadata e Navigazione

Ottiene informazioni complete sul file (dimensione, permessi, date). Legge dall'inode.

DIR *opendir(const char *name);
Apre una directory per la navigazione. Crea un context per readdir().

struct dirent *readdir(DIR *dirp);
Legge la prossima entry della directory. Restituisce nome file e tipo.

üíΩ Gestione Filesystem

Monta un dispositivo nel filesystem tree. Il VFS registra il nuovo filesystem.

int umount(const char *target);
Smonta un filesystem. Il VFS rimuove tutti i riferimenti e invalida le cache.
