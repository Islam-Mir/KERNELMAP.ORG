üíª Simulazione: Cambio di processo nel kernel

‚öôÔ∏è Processo A (bash)

üîÑ

Evento scatenante: Timer interrupt (HZ del kernel tipicamente 250-1000Hz, 1-4ms)

üîÑ Sequenza del context switch:

  1. 1. Interrupt handler: CPU salva stato corrente in thread_info
  2. 2. Scheduler invocation: schedule() seleziona prossimo processo
  3. 3. Context save: Salva registri e stack pointer in task_struct
  4. 4. Memory switch: Aggiorna il registro CR3 (x86) o equivalente per puntare alle nuove page tables dello spazio di indirizzamento
  5. 5. Context restore: Carica registri e stack del nuovo processo
  6. 6. Resume execution: Nuovo processo continua dall'istruzione successiva
üéØ Risultato: L'utente percepisce multitasking continuo, ma il kernel alterna rapidamente tra processi usando le strutture dati per salvare e ripristinare lo stato!
