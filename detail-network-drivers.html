üìã Esempi Pratici

Quando navighi in internet, ecco cosa succede nei network driver:

üîç Trasmissione Pacchetto

1. Applicazione invia dati via socket

2. Network stack aggiunge header TCP/IP/Ethernet

3. Driver riceve sk_buff dal stack

4. Mappa buffer in memoria DMA

5. Programma descriptor TX nel controller

6. Controller trasmette via DMA

7. Interrupt segnala completion

üéØ Concetti Chiave da Ricordare

‚ö°

NAPI Architecture

New API for Packet I/O rivoluziona la gestione pacchetti ad alta velocit√†. Invece di interrupt per ogni pacchetto, usa polling intelligente che riduce overhead CPU mantenendo bassa latenza.

üîÑ

Hardware Offloading

I driver moderni scaricano operazioni CPU-intensive sull'hardware: checksum calculation, TCP segmentation, interrupt coalescing. Questo permette 10Gbps+ con impatto minimo sulla CPU host.

üìä

Ring Buffers

TX e RX ring buffers sono code circolari che separano produttore (stack rete) da consumatore (hardware). Consentono operazioni asincrone e prevengono perdita pacchetti sotto carico elevato.

üîß

Device Operations

net_device_ops definisce l'interfaccia standard tra kernel e driver. Funzioni come ndo_start_xmit e ndo_open permettono al network stack di controllare l'hardware indipendentemente dal vendor.
