üìã Esempi Pratici

Quando colleghi un nuovo disco al sistema, ecco cosa succede nelle storage headers:

üîç Rilevamento Dispositivo

1. Il kernel rileva il nuovo dispositivo attraverso il bus (PCI, USB, SATA)

2. Crea una struttura block_device con le informazioni base del dispositivo

3. Legge la partition table (MBR o GPT) per identificare le partizioni

4. Per ogni partizione, determina il tipo di filesystem dal superblock

üìÑ Strutture Dati Chiave

  • struct block_device - Rappresenta un dispositivo a blocchi con geometria, dimensione e parametri operativi
  • struct hd_struct - Descrive una partizione con inizio, dimensione e numero di settore
  • struct super_block - Contiene metadati globali del filesystem: dimensione, blocchi liberi, parametri di mount
  • struct inode - Rappresenta file e directory con metadati, permessi e puntatori ai dati

üíª Nel Computer: Come Vederlo?

Comandi Linux per esplorare le storage headers:

  • fdisk -l /dev/sda - Mostra la partition table del disco
  • blkid /dev/sda1 - Visualizza informazioni sul filesystem della partizione
  • tune2fs -l /dev/sda1 - Mostra il superblock di un filesystem ext
  • lsblk -f - Elenca dispositivi e filesystem con UUID e LABEL
  • parted /dev/sda print - Mostra dettagliata tabella partizioni

üéØ Caso Reale: Montaggio Disco

Quando fai mount /dev/sdb1 /mnt:

  1. Il kernel valida che /dev/sdb1 √® una partizione valida
  2. Legge il superblock per determinare il tipo di filesystem
  3. Alloca strutture inode e dentry per la radice del filesystem
  4. Crea il punto di mount nella directory tree del VFS
  5. Il filesystem diventa accessibile attraverso /mnt

  • BTRFS superblock - Filesystem moderno con copy-on-write e snapshot
  • XFS superblock - High-performance con allocation dinamica
