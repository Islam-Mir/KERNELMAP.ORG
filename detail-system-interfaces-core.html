üìã Esempi Pratici

Quando colleghi una chiavetta USB, ecco cosa succede nelle interfacce core:

üîç Device Discovery

1. Hardware USB rileva dispositivo e genera interrupt

2. USB core riceve interrupt e avvia enumerazione

3. Device descriptor viene letto via control transfer

4. Viene creato struct device nel device model

5. Driver appropriato viene associato al dispositivo

6. Device node viene creato in /dev/

üìÑ Strutture Dati Chiave

  • struct kobject - Base per tutti gli oggetti del kernel
  • struct device - Rappresenta un dispositivo
  • struct device_driver - Driver associato al dispositivo
  • struct bus_type - Tipo di bus (PCI, USB, etc.)

üîç Esempi di Utilizzo Quotidiano

Quando installi un nuovo dispositivo hardware:

  1. Kernel rileva dispositivo sul bus
  2. device_register() crea struct device
  3. driver_register() associa driver appropriato
  4. request_irq() alloca interrupt line
  5. request_resource() riserva I/O ports
  6. Device diventa accessibile al sistema

Tutto questo avviene automaticamente grazie alle interfacce core del sistema!
