üîç Cache Hit vs Cache Miss

1. Il kernel cerca la pagina richiesta nella page cache usando l'address_space

2. Se trova la pagina (cache hit), la restituisce immediatamente dalla RAM

3. Se non trova (cache miss), genera un page fault e carica la pagina dal disco

4. La pagina caricata viene aggiunta alla cache per accessi futuri

üìÑ Strutture Dati Chiave

  • page flags - Bit che indicano stato (dirty, uptodate, locked, etc.)

üíæ Tipi di Page Cache

  • File-backed cache - Cache per file regolari mappati su disco
  • Anonymous pages - Pagine di memoria senza file sottostante (heap, stack)
  • Shared memory - Pagine condivise tra processi (System V, POSIX)
  • Memory-mapped files - File mappati direttamente in memoria virtuale
  • Buffer cache - Cache per metadata del filesystem (deprecata)

üîÑ Cosa succede dietro le quinte:

Cache Hit/Miss

Cache hit: dati in RAM, accesso istantaneo. Cache miss: page fault, caricamento da disco. Il rapporto hit/miss determina le prestazioni.
