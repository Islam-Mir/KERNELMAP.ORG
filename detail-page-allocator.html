üéØ Caso Reale: Out of Memory

Quando il sistema esaurisce pagine libere:

  1. Page allocator non trova pagine in zona richiesta
  2. Prova fallback in altre zone
  3. Se ancora fallisce, attiva page reclaim
  4. kswapd recupera pagine da cache/pagecache
  5. Se memoria ancora insufficiente, OOM killer

üîß Watermarks

  • low - Soglia bassa, attiva kswapd per reclaim
  • high - Soglia alta, kswapd smette di reclaimare
  • free - Numero totale pagine libere nella zona

üîÑ Cosa succede nell'allocazione pagine:

  1. 1. Determinazione zona: GFP_KERNEL specifica ZONE_NORMAL come zona preferita
  2. 2. Controllo watermarks: Verifica che free > min per permettere allocazione
  3. 3. Per-CPU cache: Controlla cache locale CPU per pagine "hot"
  4. 4. Buddy allocation: Richiede blocco da 4 pagine (2^2) dal buddy system
  5. 5. Page initialization: Imposta flags (_refcount=1, PG_reserved se necessario)
  6. 6. Return struct page: Restituisce puntatore alla prima pagina del blocco
  7. 7. Virtual mapping: Se necessario, mappa pagine in spazio kernel
  8. 8. Success notification: Allocazione completata con successo
