DESCRIZIONE:

Le Network Interfaces gestiscono la comunicazione diretta tra il sistema operativo e i dispositivi di rete hardware, fornendo l'astrazione necessaria per protocolli e configurazioni di rete.

CONFRONTO:

Come le porte di un aeroporto: ogni gate (interfaccia) collega l'edificio (sistema operativo) agli aerei (dispositivi di rete), gestendo arrivi, partenze e trasferimenti di passeggeri (pacchetti).

üîç Descrizione Tecnica

Le Network Interfaces sono il ponte tra il networking stack del kernel e i dispositivi hardware di rete, gestendo protocolli, indirizzi, e configurazione delle connessioni. Immagina un sistema di porti marittimi digitali: ogni banchina (interfaccia) gestisce navi (pacchetti) di diverse dimensioni e destinazioni, con gru (driver) che caricano e scaricano merci (dati) in modo sicuro ed efficiente.

‚Ä¢ Socket layer abstraction - Interfaccia uniforme per tutti i tipi di socket (TCP, UDP, RAW)

‚Ä¢ Protocol handlers - Gestione protocolli di rete (IPv4, IPv6, ARP, ICMP)

‚Ä¢ Address configuration - Gestione indirizzi IP, subnet mask, gateway

‚Ä¢ Interface management - Controllo stato interfacce (up/down, MTU, flags)

‚Ä¢ Multicast support - Gestione gruppi multicast e broadcast

‚Ä¢ VLAN support - Configurazione Virtual LAN per segmentazione rete

‚Ä¢ Interazione: Collega socket API con device drivers attraverso protocol stack

üìã Esempi Pratici

Quando invii un ping a un server remoto, ecco cosa succede nelle network interfaces:

üîç Invio Pacchetto ICMP

1. Il comando ping crea socket ICMP e prepara pacchetto echo request

2. Il kernel instrada il pacchetto attraverso la network interface appropriata

3. Il driver hardware trasmette il pacchetto sulla rete fisica

4. La network interface riceve la risposta echo reply

üìÑ Strutture Dati Chiave

  • struct net_device - Rappresenta un'interfaccia di rete con indirizzo MAC, MTU, flags di stato
  • struct sk_buff - Buffer per pacchetti di rete con headers e payload
  • struct net_device_ops - Operazioni del driver (ndo_start_xmit, ndo_open, ndo_stop)
  • struct netif_queue - Code di trasmissione per gestire congestione

üéØ Caso Reale: Download File via HTTP

Quando scarichi un file con wget:

  1. Browser risolve DNS e stabilisce connessione TCP
  2. Invia richiesta HTTP attraverso la network interface
  3. Il driver hardware trasmette pacchetti sulla rete
  4. Riceve risposta con dati del file
  5. Network interface gestisce ricezione e passaggio allo stack TCP/IP

üåê Tipi di Network Interfaces

  • Ethernet (eth0) - Interfacce wired standard (RJ45, fibra ottica)
  • Wireless (wlan0) - Interfacce Wi-Fi con supporto 802.11
  • Loopback (lo) - Interfaccia virtuale per comunicazione locale
  • VLAN (eth0.100) - Virtual LAN per segmentazione logica
  • Bridge (br0) - Bridge per connessione reti virtuali

‚ú® Caratteristiche

  • MTU Management - Maximum Transmission Unit per ottimizzare dimensione pacchetti
  • Queue Management - Code di trasmissione per gestire congestione
  • Offloading - Scarico operazioni CPU su hardware (checksum, TSO)
  • Multiqueue - Multiple code per parallelismo e prestazioni
  • Promiscuous Mode - Ricezione di tutti i pacchetti per sniffing
