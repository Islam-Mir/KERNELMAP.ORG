Storage di rete

üíæ STORAGE RETE

DESCRIZIONE:

Il Network Storage subsystem gestisce l'accesso allo storage di rete nel kernel Linux, implementando protocolli come NFS, CIFS/SMB, iSCSI e altri sistemi di storage distribuito. Coordina la comunicazione tra client e server di storage attraverso la rete.

CONFRONTO:

Immagina il Network Storage come un magazzino remoto: invece di avere gli scatoloni (dati) nel tuo garage, li tieni in un deposito esterno accessibile via strada (rete). Puoi andare a prendere quello che ti serve quando vuoi, ma devi considerare il traffico e i tempi di consegna.

üìã Esempi Pratici del Network Storage

Il Network Storage opera in ogni accesso a file remoti:

üîç Montaggio NFS

1. Client invia richiesta MOUNT al server NFS

2. Server risponde con handle del filesystem

3. Client crea superblock NFS e dentry cache

4. File remoti diventano accessibili localmente

üìÑ Strutture Dati Chiave

  • struct nfs_server - Gestisce connessione a server NFS, include indirizzo, versione protocollo, mount options
  • struct cifs_sb_info - Informazioni superblock per share CIFS, contiene credenziali e opzioni mount
  • struct iscsi_session - Sessione iSCSI con parametri connessione, autenticazione, timeout
  • struct nfs_inode - Estende inode standard con attributi NFS (filehandle, cache timeout)
  • struct cifsFileInfo - Informazioni file CIFS aperto, include handle server e lock status

üíª Comandi Linux Essenziali

  • $ mount -t nfs server:/share /mnt/nfs - Monta share NFS
  • $ mount -t cifs //server/share /mnt/smb -o username=user - Monta share SMB/CIFS
  • $ showmount -e server - Mostra export NFS disponibili
  • $ smbclient -L server - Lista share SMB disponibili
  • $ iscsiadm -m discovery -t sendtargets -p server - Scopre target iSCSI

üéØ Caso Reale: Accesso File Remoto

Quando fai $ cat /mnt/nfs/remotefile.txt:

  1. VFS inoltra richiesta a NFS client
  2. NFS client invia RPC GETATTR al server
  3. Server risponde con attributi file
  4. NFS client richiede READ con offset
  5. Dati trasferiti via rete e cachati localmente

üõ†Ô∏è API del Network Storage

Le principali funzioni per lavorare con lo storage di rete:

üîó NFS Client API

struct nfs_server *nfs_create_server(struct nfs_mount_info *mount_info);
Crea connessione a server NFS con parametri di mount.

int nfs_getattr(struct vfsmount *mnt, struct dentry *dentry, struct kstat *stat);
Ottiene attributi file dal server NFS remoto.

int nfs_open(struct inode *inode, struct file *filp);
Apre file NFS remoto e stabilisce connessione dati.

ssize_t nfs_file_read(struct file *filp, char __user *buf, size_t count, loff_t *pos);
Legge dati da file NFS con caching intelligente.

üíº CIFS/SMB API

Ottiene informazioni superblock per share CIFS.

int cifs_mount(struct cifs_sb_info *cifs_sb, struct smb_vol *vol);
Monta share SMB con autenticazione e opzioni.

int cifs_reconnect(struct TCP_Server_Info *server);
Riconnette sessioni SMB interrotte automaticamente.

üíΩ iSCSI API

struct iscsi_session *iscsi_create_session(struct Scsi_Host *shost, struct iscsi_transport *transport, int initial_cmdsn);
Crea sessione iSCSI per accesso block storage remoto.

Invia comandi SCSI incapsulati via rete iSCSI.

int iscsi_conn_bind(struct iscsi_session *session, struct iscsi_conn *conn, int is_leading);
Associa connessione TCP a sessione iSCSI.
