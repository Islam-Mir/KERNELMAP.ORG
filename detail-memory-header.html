üîÑ Cosa succede nell'header durante il ciclo di vita:

Ogni pagina ha un set di flags che indicano il suo stato: locked per I/O, dirty per modifiche, referenced per accessi recenti. Questi flags guidano tutte le decisioni di gestione memoria.

Il reference count traccia quanti "proprietari" ha una pagina. Quando arriva a zero, la pagina pu√≤ essere liberata. Questo previene liberazioni premature e memory leaks.

üó∫Ô∏è

La memoria √® divisa in zone (DMA, NORMAL, HIGHMEM) per riflettere limitazioni hardware. Ogni zona ha le sue politiche di allocazione e gestione.

LRU Management

Le pagine sono organizzate in liste LRU (Least Recently Used) per identificare candidate all'eviction. Gli header contengono puntatori per questa gestione efficiente.
